## 【重要】第9章：APIキーの管理と取り扱い

### これはAI経営本部を構築した実体験に基づく最重要事項です

### 9-1. APIキーとは何か

APIキーとは、AIサービス（Claude・Gemini・ChatGPT）にプログラムからアクセスするための「鍵」です。

重要な事実：
・APIキーが漏洩した場合、第三者があなたの費用でAIを無制限に使用できる
・気づいた時には数十万円〜数百万円の請求が来る可能性がある
・GitHubのコードに誤ってAPIキーを含めて公開してしまい、数時間で数百万円の請求が発生した事例が世界中で報告されている（GitHub Security Advisory・各社公式報告より）

### 9-2. APIキーの絶対ルール

絶対にやってはいけないこと：
1. コードの中にAPIキーを直接書く（例：api_key = "sk-proj-XXXXX"）
2. GitHubのリポジトリにAPIキーを含むファイルをアップロードする
3. メモ帳・Notion・SlackなどにそのままAPIキーを保存する
4. 他人に見せる・送信する
5. 同じAPIキーを複数のプロジェクトで使い回す

正しいAPIキーの管理方法：

方法1：環境変数として保存する（推奨）
Macのターミナルで以下を実行：
echo 'export GEMINI_API_KEY="あなたのキー"' >> ~/.zshrc
echo 'export OPENAI_API_KEY="あなたのキー"' >> ~/.zshrc
source ~/.zshrc

コードの中では以下のように呼び出す：
import os
api_key = os.environ.get("GEMINI_API_KEY")

これにより：
・コードにキーが含まれない
・GitHubにアップロードしても安全
・~/.zshrcはGitHubに含まれない

方法2：.gitignoreで保護する
AI経営本部フォルダの.gitignoreに以下が含まれていることを必ず確認：
.env
*_secret.txt
*_key.txt
setup_api.sh

### 9-3. APIキーが漏洩した時の対処法

気づいた瞬間に以下を実行：
1. 即座にAPIキーを無効化する
   ・Google AI Studio → APIキー管理 → 削除
   ・OpenAI Platform → API keys → Revoke
   ・Anthropic Console → API Keys → Delete
2. 新しいAPIキーを発行する
3. 各サービスのUsageページで使用量の異常がないか確認する
4. 各サービスのBillingページで予想外の課金がないか確認する

対処の速度が損害を最小化します。気づいたら1分以内に無効化することが重要です。

## 【重要】第10章：AIのメモリ・記憶機能とトークンコストの管理

### これを知らずに実装すると毎月数十万円の請求が来る可能性があります

### 10-1. トークンとは何か

トークン = AIが処理する文字・単語の単位（日本語約1文字 ≒ 1〜3トークン）

課金の仕組み：
入力トークン数 × 単価 ＋ 出力トークン数 × 単価 ＝ 1回の処理コスト

参考単価（2026年3月現在）：
・Claude Sonnet：入力$3/百万トークン・出力$15/百万トークン
・Gemini 2.5 Flash：入力$0.075/百万トークン
・GPT-4o mini：入力$0.15/百万トークン

### 10-2. メモリ・記憶機能が危険な理由

問題の構造：
AIシステムを構築する時に「AIに会話の記憶を持たせたい」と考えて全ての過去の会話履歴をプロンプトに含める設計にすると以下のことが起きます。

1日目：100トークンの履歴 × 10回 = 1,000トークン
1週間後：700トークンの履歴 × 10回 = 7,000トークン
1ヶ月後：3,000トークンの履歴 × 10回 = 30,000トークン
→ 月間コストが30倍に膨れ上がる

実際の被害事例：
AI経営本部を構築した際の実体験として、全会話履歴をコンテキストに含める設計にすると月間のAPI費用が想定の10〜30倍になることが確認されている。

### 10-3. 正しいメモリ管理の設計

方法1：SKILLファイル方式（推奨）
AIの「記憶」はファイルとして保存する。毎回の会話には含めない。

真田孔明のAI経営本部での実装例：
・SKILL_メルマガ自動生成.md（文体・ルール）
・SKILL_真田孔明学習メモ.md（人物像・哲学）
・SKILL_品質チェック.md（チェック基準）
→ 必要な時だけ読み込む
→ 毎回全て送信しない

方法2：直近N件のみを履歴として使用する
全履歴ではなく直近5件のみをコンテキストに含める設計にする。

方法3：Claude Codeの活用（最重要）
Claude Codeを使う場合はCLAUDE.mdがAIの「記憶」として機能する。
→ そのフォルダのコンテキストを定義
→ 毎回の会話で自動的に参照される
→ 会話履歴を蓄積しない
→ コストを最小化できる

これが真田孔明のAI経営本部がコストを抑えながら大量の処理を実行できている理由です。

### 10-4. コスト管理の実践

毎月のAPIコストを把握する方法：
・Gemini：https://console.cloud.google.com/billing
・OpenAI：https://platform.openai.com/usage
・Anthropic：https://console.anthropic.com/usage

コスト管理の目安：
・月額予算を設定してアラートを設定する
・各サービスに月額上限（Spending limit）を設定する
・急激なコスト増加は不正使用の可能性がある

### 10-5. まとめ：APIとコストの管理10の鉄則

1. APIキーは環境変数（~/.zshrc）で管理する
2. APIキーをコードに直接書かない
3. .gitignoreでAPIキー関連ファイルを除外する
4. 月額上限（Spending limit）を必ず設定する
5. AIのメモリは会話履歴ではなくSKILLファイルで管理する
6. 全履歴をプロンプトに含める設計は避ける
7. 直近N件のみを使う・または要約する
8. 毎月のAPI使用量を確認する習慣をつける
9. 異常なコスト増加を検知したら即座に調査する
10. APIキー漏洩に気づいたら1分以内に無効化する

これらを守ることで安全かつ低コストでAI経営本部を運営できます。
