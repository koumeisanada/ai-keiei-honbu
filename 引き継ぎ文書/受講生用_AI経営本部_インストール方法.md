# AI経営本部 インストール方法
# 受講生向け完全ガイド

---

## 【全体の流れ】

ステップ1: everything-claude-code導入（システムレベル）
ステップ2: 基本Introduction作成（プロジェクトレベル）
ステップ3: 新しいChatで起動テスト

---

## 【ステップ1: everything-claude-code導入】

### これは何？
AIの行動ルールを強制する仕組み
Chat内完結を技術的に防止

### やること
講義で配布される「everything-claude-code導入マニュアル」に従ってください。

すでに導入済みの方はスキップしてOK。

---

## 【ステップ2: 基本Introduction作成】

### 2-1: テンプレートを開く

場所：
~/Desktop/AI経営本部/引き継ぎ文書/
受講生用_AI経営本部_基本Introductionテンプレート.md

1. Finderを開く
2. デスクトップ → AI経営本部 → 引き継ぎ文書
3. 受講生用_AI経営本部_基本Introductionテンプレート.md を開く

### 2-2: カスタマイズする

以下の【　】部分を、あなたのビジネスに合わせて書き換えます：

必須項目（必ず書き換える）
- あなたの事業名・プロジェクト名
- 業務1〜業務6
- 対象顧客・クライアント
- 最終目標
- プロジェクトのルートフォルダ

推奨項目（できれば書き換える）
- AI-A（担当名）〜AI-C（担当名）
- 一人称、語尾、呼びかけ
- 業界、専門用語

オプション項目（使う場合のみ）
- GitHubリポジトリ名
- 参照すべき資料

### 2-3: 保存する

1. 書き換えたファイルを「別名で保存」
2. ファイル名：AI経営本部_基本Introduction_あなたの名前.md
3. 保存先：デスクトップなど、見つけやすい場所

## 【ステップ3: 新しいChatで起動テスト】

### 3-1: 新しいChatを開く

1. ブラウザでclaude.aiを開く
2. 左上の「+」ボタンをクリック
3. 新しいChat画面が開く

### 3-2: Introductionを貼り付ける

1. 保存したファイルを開く
2. 全文をコピー（Cmd+A → Cmd+C）
3. Chatのメッセージ入力欄に貼り付け（Cmd+V）
4. Enterキーを押す

### 3-3: 動作確認

AIが以下のように返信すれば成功：

了解しました。【あなたの事業名】のAI経営本部として稼働します。
何から始めますか？

### 3-4: テストコマンド

以下を入力してテスト：

テスト：簡単な企画書を作成してください

AIが以下のように返せば成功：

❌ この操作はChat内で完結できません

【Claude Codeに貼り付けてください】
```bash
（コマンド）
```

## 【よくある質問】

### Q1: 毎回貼り付けが必要？

A: はい。新しいChatを開くたびに必要です。

理由：
- Chatは会話のセッション
- 新しいChat = まっさらな状態
- だから毎回Introductionが必要

### Q2: もっと簡単にできない？

A: ファイルとして保存してアップロードする方法もあります。

1. Introductionを.txtファイルで保存
2. 新しいChatで📎ボタンをクリック
3. ファイルをアップロード
4. このファイルの内容に従ってくださいと指示

### Q3: everything-claude-codeとの違いは？

A: レイヤーが違います。

レイヤー1（システムレベル）
everything-claude-code
→ ~/.claude/ に保存（自動適用）
→ すべてのChatで有効

レイヤー2（プロジェクトレベル）
基本Introduction
→ 各Chatで貼り付け
→ そのChatでのみ有効

両方必要です。

### Q4: カスタマイズが難しい

A: 最小限の項目だけ書き換えればOKです。

必須項目（5つだけ）：
1. 事業名
2. 業務リスト
3. 対象顧客
4. 目標
5. プロジェクトフォルダ

これだけ書き換えれば使えます。

## 【トラブルシューティング】

### 問題1: AIが了解しましたと言わない

原因：Introductionが正しく貼り付けられていない

解決策：
1. もう一度全文をコピー
2. 新しいChatを開く
3. 貼り付け直す

### 問題2: Chat内でファイルを作ろうとする

原因：everything-claude-codeが導入されていない

解決策：
1. everything-claude-code導入マニュアルを確認
2. ~/.claude/ の設定を確認

### 問題3: カスタマイズ箇所が分からない

原因：【　】が見つからない

解決策：
1. テキストエディタで【を検索
2. すべての【　】を書き換える
3. 【　】がなくなるまで繰り返す
