## 【最初に知っておくべき残酷な事実】

AIには「記憶がない」。

これが全ての出発点です。

ChatGPTでもClaude でも、
チャットを閉じた瞬間に、
あなたとの会話は全て消えます。

「先週教えたこと」は覚えていません。
「昨日の続き」もありません。
「あなたの会社の事情」も、毎回ゼロから説明が必要です。

これがAIの現実です。

---

## 【人間の従業員とAIの決定的な違い】

### 人間の新入社員の場合
```
入社初日：何も知らない
    ↓
1ヶ月後：仕事の流れが分かる
    ↓
3ヶ月後：会社の文化・ルールが身につく
    ↓
1年後：ベテランとして独立して動ける
    ↓
3年後：会社の財産となる人材になる
```

人間は学習した内容が「脳」に蓄積される。
次の日も、来週も、来年も覚えている。

### AIの場合（何もしなければ）
```
月曜日：あなたの会社・事業・文体を教える
    ↓
月曜夜：チャットを閉じる
    ↓
火曜日：また最初から全部説明し直し
    ↓
水曜日：また最初から全部説明し直し
    ↓
ずっとこれが続く
```

**毎回ゼロリセット。これでは「道具」であって「社員」ではない。**

## 【スキル・知識保存の意味】

スキルファイルとGitHubへの保存は、
AIに「長期記憶」を人工的に作る行為です。
```
あなたが教える
    ↓
SKILLファイルに書き込む
    ↓
GitHubに保存（永続化）
    ↓
次のChatでSKILLファイルを読み込む
    ↓
AIが「あなたの会社のことを知っている状態」で会話できる
```

これによって初めて、
「毎回ゼロから教える作業」が不要になります。

---

## 【具体例で理解する】

### スキル保存なし（ほとんどの人がこの状態）
```
月曜：「私の会社はECサイトで物販をしています。
      文体は○○で、ターゲットは○○で...」
      （30分かけて説明）→ AIが回答

火曜：また最初から同じ説明
水曜：また最初から同じ説明
木曜：また最初から同じ説明
...

1年後：説明だけで何時間を無駄にしたか？
```

### スキル保存あり（AI経営本部の状態）
```
月曜：SKILLファイル読み込み1秒
     → AIが即座に「孔明さんの事業全体を把握した状態」で回答

火曜：同じ1秒
水曜：同じ1秒
...

修正が発生したら：SKILLファイルを更新→GitHub保存
1年後：AIが1年分の「学習」を持ち続けている
```

## 【AI経営本部として成立するための3条件】

### 条件1：知識の永続化（GitHubに保存）
- ChatやCowork上の一時保存は「消える」
- GitHubに入って初めて「永続する資産」になる
- 「GitHubに記録されていないことはやっていない」

### 条件2：知識の更新（蓄積型運用）
- 最初は完璧でなくていい
- 使うたびに修正・追記する
- 修正履歴（EDIT_LOG）を蓄積することで精度が上がり続ける

### 条件3：知識の参照（呼び出す仕組み）
- SKILLファイルが適切な場所にあること
- 必要な時に自動で参照される仕組み
- 「なぜそのルールがあるか」をファイルに記録する

## 【SKILLファイルとは何か？】

SKILLファイルとは、
AIへの「入社研修マニュアル」です。

人間の新入社員には「会社概要・業務マニュアル・行動指針」を渡します。
AIに対しても同じことをする。

それがSKILLファイルです。

### SKILLファイルに書くべき内容
```
あなたの事業概要・ターゲット顧客
文体・トーン・禁止表現
よく使うフォーマット・テンプレート
過去の成功パターン・失敗パターン
重要な数字・データ・固有名詞
```

---

## 【GitHubとは何か？（非エンジニア向け）】

GitHubは「変更履歴付きのクラウド保管庫」です。

例えるなら、
**「全ての更新履歴が残る、永遠に消えないDropbox」**です。

なぜGitHubが必要か？
```
Claude Chat上のファイル → セッションが終わると消える可能性がある
Macのデスクトップ → Macが壊れたら消える
GitHub → 世界中のサーバーに分散保存・絶対に消えない
```

**GitHubに入って初めて「資産」になる。**

## 【成功シンドロームOSとの対応】

このスキル保存プロセスは、
成功シンドロームOSの5ステップそのものです。

| OSのステップ | AI経営本部での行動 |
|---|---|
| 1.目的・願望の明確化 | 「AIに何をさせたいか」を定義する |
| 2.情報収集の徹底 | 自社の知識・ルール・事情を整理する |
| 3.分析する | どのSKILLが必要かを考える |
| 4.設計図の構築 | SKILLファイルの構造を設計する |
| 5.逆算して実行 | 毎回の修正→保存→GitHubのサイクルを回す |

## 【やらないとどうなるか？】

### 1年後の比較

#### スキル保存をサボった人
```
AIは1年前と全く同じレベルのまま
毎回ゼロから説明する時間が無駄に積み上がる
「やっぱりAIって使えないな」という結論になる
1年間で費やした説明時間：推定100〜200時間
```

#### スキル保存を続けた人
```
AIが自社の事情を完全に把握している
指示が短くなり、回答の精度が上がり続ける
「まるで長年の社員みたい」という状態になる
1年間で節約できた時間：推定100〜200時間
→ その時間を創造的な仕事に使える
```

**同じ道具を使っていても、蓄積するかしないかで、1年後に100〜200時間の差が生まれる。**

## 【AI経営本部の「蓄積型運用」サイクル】
```
Step 1：今日のAI作業で「改善点・ルール」を発見する
    ↓
Step 2：SKILLファイルに反映する
    ↓
Step 3：EDIT_LOGに「何を・なぜ・どう変えたか」を記録する
    ↓
Step 4：GitHubにプッシュして永続化する
    ↓
Step 5：次回からAIが改善後のルールで動く
    ↓
（Step 1に戻る）
```

このサイクルを回し続けることが、
AI経営本部を「資産」にする唯一の方法です。
